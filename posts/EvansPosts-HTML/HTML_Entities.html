📄 HTML Entities – Displaying Special Characters

📅 September 29, 2025

✍️ By Evans

Hey there, friends! 👋

So, one day I tried to write a simple sentence in my HTML file that said: 


<p>5 < 10 and 10 > 5</p>

…and boom 💥 — everything after the < disappeared!

That's when I learned about something called HTML Entities — a way to show special symbols like <, >, ©, or even emojis 😄 without confusing the browser.

Let’s talk about how they work.

💡 What Are HTML Entities?

HTML entities are special codes that tell the browser to display characters that would otherwise be treated as part of the HTML syntax.

They usually start with an ampersand (&) and end with a semicolon (;).

Example:


<p>5 &lt; 10 and 10 &gt; 5</p>

This will show correctly as:

5 < 10 and 10 > 5

Pretty neat, right?

🧩 Why We Need Them

The problem is, characters like <, >, &, and " have special meanings in HTML. If you type them directly, the browser might think you’re starting or ending a tag.

HTML entities help you display those symbols as plain text, not code.

🔠 Common HTML Entities I Use

Here are some of the most common ones I’ve learned:

Symbol Entity Name Example Code Result
< &lt; &lt; <
> &gt; &gt; >
& &amp; &amp; &
" &quot; &quot; "
' &apos; &apos; '
© &copy; &copy; ©
® &reg; &reg; ®
→ &rarr; &rarr; →
← &larr; &larr; ←
₦ &#8358; &#8358; ₦

🧠 Numeric Entities

Apart from named entities like < or ©, you can also use numeric entities that start with &# followed by a number.

For example:


<p>Smiley: &#128512;</p>

This will display 😀

So yes — you can even show emojis with HTML entities! 😎

🛠️ When to Use Them

  • When you want to display code snippets like <html> or <body>.
  • When your page includes currency symbols or special punctuation.
  • When characters might break the HTML syntax.

🎯 What I Learned

At first, HTML entities looked strange — but now I see them as tiny secret codes that keep my pages from breaking.

They make your HTML cleaner, safer, and more professional, especially when displaying code or special symbols.

Thanks for reading — and if you’ve ever wondered why your < or > disappear, now you know the trick! 😉
